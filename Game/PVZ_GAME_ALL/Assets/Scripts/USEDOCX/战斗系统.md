# PVZ 战斗系统技术实现文档（README）

## 1. 项目概述
本项目为 Unity 实现的“植物大战僵尸”战斗系统，目标是搭建一个结构清晰、可扩展、配置驱动的战斗框架，便于快速迭代植物、僵尸、关卡与玩法。

## 2. 文档目标与范围
- 目标：明确战斗系统整体架构、核心流程、模块职责、关键数据结构、可扩展点与性能策略。
- 范围：战斗相关（种植、僵尸、攻击、子弹、状态、结算、波次/关卡）。
- 非范围：美术制作流程、关卡编辑器具体实现、完整 UI 皮肤工程。

## 3. 总体架构与程序集划分
系统采用模块化分层结构，通过程序集与命名空间解耦。各模块职责如下：

- GameBase：系统接口定义与基础生命周期
- Config：配置系统与游戏数据
- SourceLoad：资源加载抽象层
- ObjectPool：对象池系统
- EventBus：事件总线
- Audio：音频管理
- UI：用户界面管理
- DataManager：数据与存档管理
- Core：系统初始化与调度

## 4. 战斗系统模块划分
战斗系统建议独立命名空间 `Battle` 或 `Gameplay`，主要包含：

- BattleController：战斗生命周期管理（Init/Start/Running/End）
- LevelRuntime：关卡运行时数据（地形、波次、规则）
- GridSystem：格子与种植规则
- UnitSystem：植物/僵尸的统一注册与管理
- WaveScheduler：波次刷新与刷怪调度
- CombatSystem：攻击、子弹、命中与伤害
- StatusSystem：状态效果（减速/冻结/燃烧）
- ResourceSystem：阳光产出、消耗与拾取
- ResultSystem：胜负判定与结算

## 5. 核心数据模型（配置驱动）
所有单位与关卡建议配置化，支持 ScriptableObject 或表格：

- PlantConfig：Cost、CD、攻击间隔、射程、子弹ID、特殊能力
- ZombieConfig：HP、移速、护甲、攻击间隔、特殊行为
- BulletConfig：飞行方式、伤害类型、范围/穿透
- LevelConfig：地图尺寸、地形、初始阳光、波次规则

### 5.1 运行时数据结构建议
- LevelRuntime：关卡配置实例化后的运行态，含地图尺寸、地形格子状态、当前波次索引、剩余僵尸数。
- GridCell：每格存放植物引用、地形标记、占用状态、特殊载体（荷叶、南瓜套）。
- UnitHandle：运行态统一句柄，记录单位类型、实例ID、当前行列、血量、状态列表。

## 6. 核心流程（思路级）
### 6.1 战斗初始化
1. Core 初始化系统
2. Config 载入关卡配置
3. BattleController 创建 LevelRuntime
4. GridSystem 生成格子
5. ObjectPool 预热资源
6. UI 展示关卡信息

### 6.2 波次刷新
1. WaveScheduler 检查触发条件
2. 从对象池生成僵尸
3. UnitSystem 注册
4. EventBus 通知 UI

### 6.3 植物种植
1. UI 选择植物进入放置态
2. GridSystem 校验格子可放置
3. ResourceSystem 消耗阳光
4. ObjectPool 生成植物并注册

### 6.4 攻击与命中
1. 植物扫描目标
2. 满足攻击间隔生成子弹
3. 子弹移动并命中
4. CombatSystem 计算伤害
5. StatusSystem 处理状态效果

### 6.5 结算
1. ResultSystem 监听僵尸数量
2. 满足胜利或失败条件
3. 切换至结算 UI

## 7. 模块实现要点

### 7.1 GridSystem
- 使用二维数组维护格子状态
- 提供世界坐标与格子索引转换
- 支持行/列快速查询
- 植物放置验证建议包含：地形类型校验、已有植物校验、阳光校验

### 7.2 UnitSystem
- 用字典或列表统一管理植物/僵尸
- 行级索引缓存，提升扫描效率
- 统一提供：注册、移除、查找、范围查询

### 7.3 CombatSystem
- 统一伤害入口
- 支持不同伤害类型与护甲规则
- 子弹与攻击逻辑解耦
- 攻击流程建议：植物攻击策略 -> 子弹策略 -> 命中策略

### 7.4 WaveScheduler
- 时间轴 + 条件混合触发
- 支持大波提示与间隔
- 波次可由配置驱动，支持“固定时间刷怪”和“条件刷怪”

### 7.5 StatusSystem
- 状态模块独立管理
- 支持覆盖/叠加/延长策略
- 状态效果统一 Tick 与结束逻辑

### 7.6 ResourceSystem
- 阳光产出：固定生成、植物产出、掉落
- 阳光消耗：种植、特殊技能
- UI 与资源系统事件解耦

### 7.7 ResultSystem
- 胜利条件：波次完成且场上无僵尸
- 失败条件：僵尸进入终点
- 结算流程与战斗逻辑分离

## 8. 事件设计建议（EventBus）
建议的战斗事件枚举：
- PlantPlaced
- PlantRemoved
- ZombieSpawned
- ZombieDead
- SunChanged
- WaveStarted
- WaveEnded
- GameWin
- GameLose

## 9. AI 与行为策略建议
- 植物：攻击节奏基于间隔计时器，扫描行内目标优先级可配置
- 僵尸：状态机驱动（移动/攻击/特殊行为）
- 特殊僵尸可通过策略组件注入行为

## 10. 伤害与护甲规则建议
- 基础伤害 = 攻击力 × 伤害类型修正
- 护甲优先级：护甲吸收 -> 本体扣血
- 状态伤害独立于普通攻击

## 11. 性能与扩展策略
- 统一对象池减少 GC
- 事件驱动减少耦合
- 行级索引减少目标扫描成本
- 状态系统独立便于扩展新植物与僵尸

## 12. 扩展点建议
- 新植物：新增配置 + 攻击策略
- 新僵尸：新增配置 + 行为扩展
- 新关卡：新增地形与波次规则

## 13. 开发规范建议
- 所有配置统一走 Config 系统
- 不直接依赖 Unity 组件，逻辑可测试
- 事件使用 EventBus，避免跨模块引用
