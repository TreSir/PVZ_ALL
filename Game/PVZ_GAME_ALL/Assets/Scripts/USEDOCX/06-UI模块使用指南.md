# UI 模块使用指南

## 概述

UI 模块负责游戏中所有界面面板的管理，包括面板的打开、关闭、层级管理和对象池化。采用**约定优于配置**设计，新增面板无需手动注册路径。

---

## 面板生命周期

```
None → Opening → Opened → Paused → Resumed → Closing → Closed
```

| 状态 | 触发条件 | 回调方法 |
|------|----------|----------|
| Opening | `OpenPanel()` | `OnBeforeOpen()` |
| Opened | 面板激活完成 | `OnOpened()` |
| Paused | `OnDisable()` | `OnPause()` |
| Resumed | `OnEnable()` | `OnResume()` |
| Closing | `ClosePanel()` | `OnBeforeClose()` |

---

## 创建新面板（3步）

### 第1步：创建脚本

```csharp
using UI;

public class SettingsPanel : BasePanel
{
    public override string UIName => "SettingsPanel";

    public override void InitUI()
    {
    }

    protected override void OnOpened()
    {
    }

    protected override void OnBeforeClose()
    {
    }
}
```

### 第2步：创建 Prefab

在 `Resources/Prefabs/UI/` 目录下创建 Prefab，**Prefab 名字必须和 UIName 一致**：

```
Resources/Prefabs/UI/SettingsPanel.prefab  ← 名字一致
```

### 第3步：打开面板

```csharp
BaseUIManager.Instance.OpenPanel("SettingsPanel");
```

**无需注册路径，直接使用！**

---

## 路径约定

### 默认约定（推荐）

面板名 = Prefab名，自动匹配：

```
OpenPanel("SettingsPanel")
  → 加载 Resources/Prefabs/UI/SettingsPanel.prefab ✅
```

### 自定义路径（特殊需求）

Prefab 在子目录下时，需要手动注册：

```csharp
BaseUIManager.Instance.RegisterCustomPath("BossPanel", "Battle/BossPanel");
// → 加载 Resources/Prefabs/UI/Battle/BossPanel.prefab
```

### 目录结构

```
Resources/
└── Prefabs/
    └── UI/
        ├── MainMenuPanel.prefab       ← 默认约定
        ├── SettingsPanel.prefab       ← 默认约定
        └── Battle/
            └── BossPanel.prefab       ← 需要注册自定义路径
```

## 面板管理

```csharp
using UI;

BaseUIManager.Instance.OpenPanel("MainMenuPanel");
BaseUIManager.Instance.ClosePanel("MainMenuPanel");
BaseUIManager.Instance.CloseAllPanel();
BaseUIManager.Instance.IsPanelOpened("MainMenuPanel");
```

## 生命周期回调

```csharp
public class SettingsPanel : BasePanel
{
    public override string UIName => "SettingsPanel";

    public override void InitUI() { }

    protected override void OnBeforeOpen() { }   // 打开前
    protected override void OnOpened() { }       // 打开后
    protected override void OnPause() { }        // 被遮挡
    protected override void OnResume() { }       // 重新显示
    protected override void OnBeforeClose() { }  // 关闭前
}
```

## 对象池

面板关闭后不会销毁，而是回收到对象池，下次打开时复用，减少 GC。

```csharp
BaseUIManager.Instance.PreloadPanel("SettingsPanel");  // 预加载
BaseUIManager.Instance.ClearPool();                     // 清空池
```

**配置：** 在 `GameSettingConfig.asset` 中设置 `enablePanelPool` 和 `panelPoolInitialSize`。

---

## 配置说明

**位置：** `Resources/Configs/GameSettingConfig.asset`

| 配置项 | 说明 |
|--------|------|
| startupPanelName | 启动面板名称 |
| enablePanelPool | 是否启用面板池 |
| panelPoolInitialSize | 面板池初始大小 |

---

## API 速查

| 方法 | 说明 |
|------|------|
| `OpenPanel(name)` | 打开面板 |
| `ClosePanel(name)` | 关闭面板 |
| `CloseAllPanel()` | 关闭所有面板 |
| `GetPanel(name)` | 获取面板实例 |
| `IsPanelOpened(name)` | 面板是否已打开 |
| `PreloadPanel(name)` | 预加载面板到池 |
| `ClearPool()` | 清空面板池 |
| `RegisterCustomPath(name, path)` | 注册自定义路径 |

---

## 注意事项

- Prefab 名字必须和 `UIName` 保持一致
- Prefab 必须放在 `Resources/Prefabs/UI/` 目录下
- Prefab 上必须挂载继承 `BasePanel` 的脚本
- 子目录下的 Prefab 需要调用 `RegisterCustomPath` 注册
